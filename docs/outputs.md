# Outputs

The workflow generates quality control (QC) reports, extracted gene sequences, alignments, phylogenetic trees, ANI analyses and logs.

## Output overview

The workflow output structure is shown below:

```text
QC/
|-- quast/
|-- gene-qc-detail.tsv
|-- gene-qc-matrix.pdf
|-- gene-qc-matrix.tsv
|-- gene-qc-summary.tsv
|-- gene-qc.log
|-- genome-list-pass.txt
\-- ref_genes.validated

genes/
|-- aligned/
|-- concat/
\-- map-pool/


phylogenetics/
|-- MLSA.nwk
|-- iqtree/
|-- raxml/
\-- fasttree/

ANI/
|-- skani/
|-- fastani/
\-- pyani/

public_genomes/

logs/
```

## Quality control outputs

The `QC/` directory contains all quality-control outputs generated before phylogenetic and ANI analyses.


### Reference gene validation

The validated FASTA file `db/ref-genes.validated.fas`, is used throughout the workflow and checks for valid header format and unique strain|gene identifiers. 

If duplicate strain|gene identifiers or invalid headers are found it gives an error in the log `QC/ref_genes.validated`. Otherwise it is reported as *OK*.

### QUAST assembly statistics

Contains assembly quality statistics generated by the tool QUAST. The most important file is: `QC/quast/report.pdf`

This report summarizes and plots assembly metrics such as:

- Total assembly length (bp)
- Number of contigs
- N50 (bp)
- GC content (%)

These results can be used to identify low-quality genome assemblies before downstream analysis.

### Gene quality control

The gene QC step evaluates extracted loci and determines whether genomes are suitable for MLSA and ANI analyses.

**Detailed QC table:**

`QC/gene-qc-detail.tsv` contains one row per sample-gene combination.

Important columns include:

| Column | Description |
|----------|-------------|
| Sample | Genome/sample name |
| Gene | Locus name |
| Copies | Number of gene copies |
| Length | Extracted sequence length (from genome) |
| RefLength | Reference gene length (from `ref-genes.fas`)|
| LengthRatio | Relative gene length |
| Coverage | Alignment coverage |
| Similarity | Alignment similarity |
| Status | QC classification |

Possible status values:

| Status | Meaning |
|----------|----------|
| OK | Gene passed QC |
| DUPLICATED | Multiple copies detected |
| FRAGMENTED | Incomplete gene detected |
| MISSING | Gene not found |

**Sample summary**

`QC/gene-qc-summary.tsv` summarizes QC results per genome.

Reported values include, number of missing loci, number of duplicated loci, number of fragmented loci and overall PASS/FAIL status for that sample.

**Presence/absence matrix**

`QC/gene-qc-matrix.tsv` provides a gene-by-genome matrix.

Values:

| Value | Meaning |
|---------|----------|
| 1 | Present |
| 2 | Duplicated |
| F | Fragmented |
| NA | Missing |

`QC/gene-qc-matrix.pdf` provides the table in neat PDF format.

**Passing genomes**

`QC/genome-list-pass.txt` lists genomes that passed all gene QC criteria and thus contain only single-copy and complete genes. Only these genomes are used for MLSA and ANI analyses.


## Extracted genes

The `genes/` directory contains extracted loci, alignments, and concatenated MLSA datasets.

**Combined gene dataset**

`genes/map-pool/map-pool.fas` contains all extracted loci.  
`genes/map-pool/map-pool.filtered.fas` contains only loci originating from genomes that passed gene QC.


**Multiple sequence alignments**

`genes/aligned/` contains one alignment per locus generated using MUSCLE.

For example:

```text
genes/aligned/actin.fas
genes/aligned/rpb2.fas
genes/aligned/tub2.fas
```

**Concatenated alignment**

`genes/concat/concat.fas` is the combined multilocus alignment used for phylogenetic inference.

**Partition information**

`genes/concat/concat.part` defines locus boundaries within the concatenated alignment.  
`genes/concat/concat.tab` is used by IQ-TREE and RAxML for partitioned analyses.

## Phylogenetic outputs

**Final tree**

`phylogenetics/MLSA.nwk` is the primary phylogenetic result of the workflow.

The tree is exported in Newick format and can be visualized with tools such as iTOL and ggtree.

### Method-specific outputs

Depending on the selected tree method, additional files are produced.

**IQ-TREE**

`phylogenetics/iqtree/` contains model selection results, bootstrap statistics, logs, and tree files generated by IQ-TREE.

**RAxML**

`phylogenetics/raxml/` contains bootstrap replicates, best trees, and bipartition trees generated by RAxML.

**FastTree**

`phylogenetics/fasttree/` contains FastTree output tree.

## ANI outputs

ANI outputs are only generated when ANI analysis is enabled.

**skani**

`ANI/skani/skani_table.tsv` contains pairwise ANI values.  
`ANI/skani/skani.pdf` combines the MLSA tree with an ANI similarity matrix.

**FastANI**

`ANI/fastani/fastani_table.tsv` contains pairwise ANI values.  
`ANI/fastani/fastani.pdf` combines the MLSA tree with an ANI similarity matrix.

**pyani**

`ANI/pyani/ANIm_percentage_identity.tab` contains the percentage identity matrices.  
`ANI/pyani/ANIm_alignment_coverage.tab` contains the alignment coverage matrices.

`ANI/pyani/pyani_percentage_identity_plot.pdf` tree-integrated visualization of percentage identity.  
`ANI/pyani/pyani_cov_plot.pdf` tree-integrated visualization of alignment coverage.

## Public genomes

When public genomes are specified in `public_genomes.txt`, downloaded assemblies are stored in `public_genomes/`.

Downloaded genomes are treated identically to local genome assemblies throughout the workflow.

## Logs

`logs/` contains log files for all workflow steps.

These files are useful for troubleshooting failed runs and inspecting tool-specific output.
