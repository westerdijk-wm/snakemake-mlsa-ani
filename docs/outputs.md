# Outputs

The workflow generates quality control reports, extracted gene sequences, alignments, phylogenetic trees, and optional ANI analyses.

Most users will primarily inspect:

- `QC/`
- `phylogenetics/`
- `ANI/`

The remaining files are intermediate results that may be useful for troubleshooting or downstream analyses.

## Quality control outputs

### Reference gene validation

```text
QC/ref_genes.validated
db/ref-genes.validated.fas
```

The reference database is validated before analysis begins.

The validated FASTA file is used throughout the workflow and guarantees:

- valid header format
- unique strain|gene identifiers
- gene names compatible with the configuration

### QUAST assembly statistics

```text
QC/quast/
```

Contains assembly quality statistics generated by QUAST.

The most important file is:

```text
QC/quast/report.pdf
```

This report summarizes assembly metrics such as:

- assembly size
- number of contigs
- N50
- GC content
- other assembly statistics

### Gene quality control

The gene QC step evaluates extracted loci and determines whether genomes are suitable for MLSA and ANI analyses.

#### Detailed QC table

```text
QC/gene-qc-detail.tsv
```

Contains one row per sample-gene combination.

Important columns include:

| Column | Description |
|----------|-------------|
| Sample | Genome/sample name |
| Gene | Locus name |
| Coverage | Alignment coverage |
| Similarity | Alignment similarity |
| Length | Extracted sequence length |
| RefLength | Reference gene length |
| LengthRatio | Relative gene length |
| Status | QC classification |

Possible status values:

| Status | Meaning |
|----------|----------|
| OK | Gene passed QC |
| DUPLICATED | Multiple copies detected |
| FRAGMENTED | Incomplete gene detected |
| MISSING | Gene not found |

#### Sample summary

```text
QC/gene-qc-summary.tsv
```

Summarizes QC results per genome.

Reported values include:

- number of missing loci
- number of duplicated loci
- number of fragmented loci
- overall PASS/FAIL status

#### Presence/absence matrix

```text
QC/gene-qc-matrix.tsv
QC/gene-qc-matrix.pdf
```

Provides a gene-by-genome matrix.

Values:

| Value | Meaning |
|---------|----------|
| 1 | Present |
| 2 | Duplicated |
| F | Fragmented |
| NA | Missing |

The PDF version provides a neat overview table.

#### Passing genomes

```text
QC/genome-list-pass.txt
```

Lists genomes that passed all gene QC criteria.

Only these genomes are used for ANI analyses.

## Extracted genes

### Individual genome mappings

```text
genes/map/
```

Contains extracted loci recovered from each genome.

Each sequence represents a putative homolog identified through reference-based mapping.

### Combined gene dataset

```text
genes/map-pool/
```

Important files:

```text
genes/map-pool/map-pool.fas
genes/map-pool/map-pool.filtered.fas
```

The filtered FASTA contains only loci from genomes that passed gene QC.

## Multiple sequence alignments

```text
genes/aligned/
```

Contains one alignment per locus.

Example:

```text
genes/aligned/actin.fas
genes/aligned/rpb2.fas
genes/aligned/tub2.fas
```

These alignments are generated using MUSCLE.

## Concatenated MLSA dataset

```text
genes/concat/
```

### Concatenated alignment

```text
genes/concat/concat.fas
```

Combined multilocus alignment used for phylogenetic inference.

### Partition information

```text
genes/concat/concat.part
genes/concat/concat.tab
```

Defines locus boundaries within the concatenated alignment.

Used by IQ-TREE and RAxML for partitioned analyses.

## Phylogenetic outputs

### Final tree

```text
phylogenetics/MLSA.nwk
```

This is the primary phylogenetic result of the workflow.

The tree is exported in Newick format and can be visualized with tools such as:

- FigTree
- iTOL
- ggtree

### Method-specific outputs

Depending on the selected tree method, additional files are produced.

#### IQ-TREE

```text
phylogenetics/iqtree/
```

Contains model selection results, bootstrap statistics, logs, and tree files generated by IQ-TREE.

#### RAxML

```text
phylogenetics/raxml/
```

Contains bootstrap replicates, best trees, and bipartition trees generated by RAxML.

#### FastTree

```text
phylogenetics/fasttree/
```

Contains FastTree output files.

## ANI outputs

ANI outputs are only generated when ANI analysis is enabled.

### skani

```text
ANI/skani/
```

Important files:

```text
ANI/skani/skani_table.tsv
ANI/skani/skani.pdf
```

The table contains pairwise ANI values.

The PDF combines the MLSA tree with an ANI similarity matrix.

### FastANI

```text
ANI/fastani/
```

Important files:

```text
ANI/fastani/fastani_table.tsv
ANI/fastani/fastani.pdf
```

The table contains pairwise ANI values.

The PDF combines the MLSA tree with an ANI similarity matrix.

### PyANI

```text
ANI/pyani/
```

Important files:

```text
ANI/pyani/ANIm_percentage_identity.tab
ANI/pyani/ANIm_alignment_coverage.tab

ANI/pyani/pyani_percentage_identity_plot.pdf
ANI/pyani/pyani_cov_plot.pdf
```

PyANI generates:

- percentage identity matrices
- alignment coverage matrices
- tree-integrated visualization plots

## Public genomes

```text
public_genomes/
```

When public genomes are specified in `public_genomes.txt`, downloaded assemblies are stored here.

Downloaded genomes are treated identically to local genome assemblies throughout the workflow.

## Logs

```text
logs/
```

Contains log files for all workflow steps.

These files are useful for troubleshooting failed runs and inspecting tool-specific output.
