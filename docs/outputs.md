# Outputs

The workflow generates quality control (QC) reports, extracted gene sequences, alignments, phylogenetic trees, ANI analyses, and logs.

## Output overview

The workflow output structure is shown below:

```text
QC/
|-- quast/
|   \-- report.pdf
|-- ref_genes.validated
|-- gene-qc-detail.tsv
|-- gene-qc-summary.tsv
|-- gene-qc-matrix.tsv
|-- gene-qc-matrix.pdf
|-- gene-qc.log
\-- genome-list-pass.txt

genes/
|-- map-pool/
|   |-- map-pool.fas
|   \-- map-pool.filtered.fas
|-- aligned/
\-- concat/
    |-- concat.fas
    |-- concat.tab
    \-- concat.part

phylogenetics/
|-- MLSA.nwk
|-- iqtree/
|-- raxml/
\-- fasttree/

ANI/
|-- skani/
|-- fastani/
\-- pyani/

public_genomes/

logs/

report.html
```

> Only the directories/files relevant to the chosen `tree.method` and `ani_method` will be generated (see [Configuration](configuration.md)).

## Quality control outputs

The `QC/` directory contains all quality-control outputs generated before phylogenetic and ANI analyses.

### Reference gene validation

`db/ref-genes.validated.fas` is the validated reference FASTA used throughout the workflow. Validation checks for a valid header format and unique `strain|gene` identifiers.

The result is written to `QC/ref_genes.validated`:

- If duplicate `strain|gene` identifiers or invalid headers are found, an error is reported.
- Otherwise, the file reports `OK`.

### QUAST assembly statistics

`QC/quast/report.pdf` contains assembly quality statistics generated by QUAST, including:

- Total assembly length (bp)
- Number of contigs
- N50 (bp)
- GC content (%)

These results can be used to identify low-quality genome assemblies before downstream analysis.

### Gene quality control

The gene QC step evaluates extracted loci and determines whether genomes are suitable for MLSA and ANI analyses.

#### Detailed QC table

`QC/gene-qc-detail.tsv` contains one row per sample-gene combination.

| Column | Description |
|---|---|
| `Sample` | Genome/sample name |
| `Gene` | Locus name |
| `Copies` | Number of gene copies |
| `Length` | Extracted sequence length (from genome) |
| `RefLength` | Reference gene length (from `ref-genes.fas`) |
| `LengthRatio` | Relative gene length |
| `Coverage` | Alignment coverage |
| `Similarity` | Alignment similarity |
| `Status` | QC classification |

Possible `Status` values:

| Status | Meaning |
|---|---|
| `OK` | Gene passed QC |
| `DUPLICATED` | Multiple copies detected |
| `FRAGMENTED` | Incomplete gene detected |
| `MISSING` | Gene not found |

#### Sample summary

`QC/gene-qc-summary.tsv` summarizes QC results per genome, including the number of missing, duplicated, and fragmented loci, and an overall PASS/FAIL status for that sample.

#### Presence/absence matrix

`QC/gene-qc-matrix.tsv` provides a gene-by-genome matrix:

| Value | Meaning |
|---|---|
| `1` | Present |
| `2` | Duplicated |
| `F` | Fragmented |
| `NA` | Missing |

`QC/gene-qc-matrix.pdf` renders this table in PDF format.

#### Passing genomes

`QC/genome-list-pass.txt` lists genomes that passed all gene QC criteria (single-copy, complete genes for all loci). Only these genomes are used for MLSA and ANI analyses.

## Extracted genes

The `genes/` directory contains extracted loci, alignments, and concatenated MLSA datasets.

### Combined gene dataset

- `genes/map-pool/map-pool.fas` — all extracted loci from all samples.
- `genes/map-pool/map-pool.filtered.fas` — loci from genomes that passed gene QC only.

### Multiple sequence alignments

`genes/aligned/` contains one alignment per locus, generated using MUSCLE, e.g.:

```text
genes/aligned/actin.fas
genes/aligned/rpb2.fas
genes/aligned/tub2.fas
```

### Concatenated alignment

`genes/concat/concat.fas` is the combined multilocus alignment used for phylogenetic inference.

### Partition information

- `genes/concat/concat.tab` — per-locus boundary information.
- `genes/concat/concat.part` — partition file used by IQ-TREE and RAxML for partitioned analyses.

## Phylogenetic outputs

### Final tree

`phylogenetics/MLSA.nwk` is the primary phylogenetic result of the workflow: a rerooted Newick tree that can be visualized with tools such as iTOL or ggtree.

### Method-specific outputs

Only the directory corresponding to the configured `tree.method` is generated.

| Method | Directory | Contents |
|---|---|---|
| IQ-TREE | `phylogenetics/iqtree/` | Model selection results, bootstrap statistics, logs, and tree files |
| RAxML | `phylogenetics/raxml/` | Bootstrap replicates, best tree, and bipartition tree |
| FastTree | `phylogenetics/fasttree/` | FastTree output tree |

## ANI outputs

ANI outputs are only generated when `ani_method` is set to something other than `none`.

### skani

- `ANI/skani/skani_table.tsv` — pairwise ANI values.
- `ANI/skani/skani.pdf` — MLSA tree combined with an ANI similarity matrix.

### FastANI

- `ANI/fastani/fastani_table.tsv` — pairwise ANI values.
- `ANI/fastani/fastani.pdf` — MLSA tree combined with an ANI similarity matrix.

### pyani

- `ANI/pyani/ANIm_percentage_identity.tab` — percentage identity matrix.
- `ANI/pyani/ANIm_alignment_coverage.tab` — alignment coverage matrix.
- `ANI/pyani/pyani_percentage_identity_plot.pdf` — tree-integrated visualization of percentage identity.
- `ANI/pyani/pyani_cov_plot.pdf` — tree-integrated visualization of alignment coverage.

---

## Public genomes

When public genomes are specified via `public_genomes` in `config.yaml`, downloaded assemblies are stored in `public_genomes/`.

Downloaded genomes are treated identically to local genome assemblies throughout the workflow.

---

## Logs

`logs/` contains log files for all workflow steps, organized by tool/rule. These are useful for troubleshooting failed runs and inspecting tool-specific output.

---

## Snakemake report

In addition to the files described above, an interactive HTML summary report can be generated after a successful run:

```bash
snakemake --report report.html
```

`report.html` is a self-contained file that includes:

- A rule graph (DAG) showing how rules and outputs are connected (see [Methods](methods.md) for an example).
- All outputs marked with `report(...)` in the workflow (e.g. `QC/quast/report.pdf`, `QC/gene-qc-matrix.pdf`, and the ANI heatmap PDF for the configured `ani_method`), embedded and viewable directly in the browser.
- Runtime statistics and configuration details for the run.

This report is useful for sharing results and provenance information without requiring access to the full output directory structure.
