# Methods

The workflow performs multilocus sequence analysis (MLSA), phylogenetic
inference, and Average Nucleotide Identity (ANI) analysis directly from
assembled genome sequences.

## Software overview

The workflow integrates established bioinformatics tools together with custom
scripts for quality control, data transformation, and visualization.

Software versions are defined through the workflow Conda environments located
in `workflow/envs/`, ensuring reproducible analyses. Only the tree-method and
ANI-method tools relevant to the active configuration (see
[Configuration](configuration.md)) are used in a given run.

## Input genomes

Genome assemblies are supplied through the `genomes/` directory in FASTA
format (`.fna`, `.fasta`, `.fas`, `.fa`). The sample name used throughout the
workflow is derived from the filename.

Optionally, additional public genomes can be downloaded directly from NCBI
using accession numbers listed in `config/public_genomes.txt`. Downloaded
assemblies are placed in `resources/public_genomes/` and incorporated into
the workflow identically to local genomes.

## Reference gene validation

Reference loci are supplied in `config/ref-genes.fas` (configurable via
`ref_genes` in `config.yaml`).

Prior to analysis, the reference database is validated using the custom script
`validate-ref-genes.py`. This step:

- verifies that every header follows the `>{strain}|{gene}` format
- normalizes whitespace and spacing around the `|` separator
- checks for duplicate `strain|gene` identifiers
- writes a cleaned copy of the database to `resources/db/ref-genes.validated.fas`

The validated database is used throughout all subsequent mapping and
quality-control steps. If validation fails, the workflow stops and reports the
offending headers in the log.

## Assembly quality assessment

Assembly quality is evaluated for every genome using
[QUAST](https://quast.sourceforge.net/) (rule `quast`).

Reported metrics include:

- Total assembly length
- Number of contigs
- N50
- GC content

These statistics provide an overview of assembly completeness and contiguity
before downstream analyses. Results are available as per-sample PDF reports
under `results/QC/quast/`.

## Reference gene mapping and locus extraction

Reference genes are mapped against each genome assembly using
[Minimap2](https://github.com/lh3/minimap2) in long-read mode
(`-x map-ont`), allowing secondary alignments (`-p 0.5`) to recover all
candidate locus hits (rule `minimap2`).

Initial SAM output is filtered in two steps:

1. Unmapped reads are removed (`samtools view -F4`), strand orientation is
   corrected (`sam-flip.pl`), and only the single best hit per locus is
   retained (`sam-keep-best.pl`; rule `sam_filter`).
2. Filtered hits are realigned against the reference gene sequences using
   MUSCLE via the custom script `sam-realign.pl`, which trims alignment
   boundaries to the nearest complete homolog region (rule `sam_realign`).

Candidate loci are extracted from the realigned SAM using
`sam-extract-hit-seq.pl`. Only hits with a minimum sequence similarity of
70% are retained (rule `sam_extract_hit_seq`).

Extracted sequences are renamed to a consistent `{sample}|{gene}` header
format using `rename-extracted-hit-fasta.pl` (rule `rename_extracted_hit_seq`).

Fragmented hits from the same locus are merged where possible using
`join-fragments.sh` (rule `join_fragments`), and exact duplicate sequences
within each sample are removed, keeping the first occurrence
(rule `genes_deduplicate`).

## Gene quality control

All per-sample locus FASTA files are pooled and evaluated using the custom
script `genes-qc.py` (rule `genes_qc`).

For each sample-gene combination, the workflow assesses:

- **Copy number** — number of hits for that locus
- **Alignment coverage** — fraction of the reference gene covered
- **Relative gene length** — extracted sequence length relative to the
  reference gene length

A hit is classified as fragmented if alignment coverage falls below 95% or
the length ratio falls below 0.90. A sample passes QC only when all target
loci are present as a single, unfragmented copy.

The workflow generates:

- A detailed per-hit QC table (`results/QC/gene-qc-detail.tsv`)
- A gene copy-number matrix across all samples (`results/QC/gene-qc-matrix.tsv`)
- A PDF visualization of the matrix (`results/QC/gene-qc-matrix.pdf`;
  rule `table_plot`)
- A list of genome file paths for all passing samples
  (`results/QC/genome-list-pass.txt`), used as input for ANI analysis

Genomes containing missing, duplicated, or fragmented loci are excluded from
all subsequent MLSA and ANI analyses.

## Multiple sequence alignment

For each locus, sequences from all passing genomes are extracted and aligned
independently using [MUSCLE](https://www.drive5.com/muscle/) (rule `align`).
This produces one alignment per gene, preserving locus-specific evolutionary
signal.

## Alignment concatenation

Individual locus alignments are combined into a single MLSA supermatrix using
`fasta_autoconcatenate` (rule `concat`). The workflow also generates partition
information describing locus boundaries within the concatenated alignment
(rule `partition_file`), used by partition-aware phylogenetic methods.

## Phylogenetic inference

Phylogenetic reconstruction is performed using one of three selectable
methods, configured via `tree.method` (see [Configuration](configuration.md)).
The resulting tree is rerooted at the midpoint using `nw_reroot` from
[Newick Utilities](https://github.com/tjunier/newick_utils) and exported as
`results/phylogenetics/MLSA.nwk` (rule `reroot_tree`).

### IQ-TREE (recommended)

[IQ-TREE](https://iqtree.github.io/) performs partitioned maximum-likelihood
inference using:

- ModelFinder Plus (`-m MFP`) for automatic model selection per partition
- Ultrafast bootstrap support estimation (`-B`)
- Bootstrap replicates: minimum 100, recommended ≥ 1000

### RAxML

[RAxML](https://cme.h-its.org/exelixis/web/software/raxml/) performs
partitioned maximum-likelihood inference using:

- GTR+GAMMA substitution model
- Rapid bootstrap analysis followed by ML tree search (`-f a`)
- Bootstrap replicates: minimum 1, recommended ≥ 100

Bootstrap support values are mapped onto the best-scoring tree to produce the
final bipartitions tree (rule `raxml_bipartitions`).

### FastTree

[FastTree](http://www.microbesonline.org/fasttree/) performs approximate
maximum-likelihood inference using:

- GTR substitution model with gamma-distributed rate heterogeneity (`-gtr -nt -gamma`)

FastTree is the fastest option but is not partition-aware and does not perform
bootstrapping. The `tree.bootstrap` setting is ignored when this method is
selected.

## Average Nucleotide Identity (ANI)

ANI analysis is optional (configured via `ani_method`) and is performed only
on genomes that passed all gene QC criteria. The genome list from
`results/QC/genome-list-pass.txt` is used as input for all ANI methods.

Pairwise output from each method is reshaped into a symmetric square identity
matrix using the custom script `ani2table.py` (rule `ani_table`) and
visualized alongside the MLSA phylogeny using `tree-ANI-heatmap.R`
(rule `ani_plot`).

### skani

[skani](https://github.com/bluenote-1577/skani) computes pairwise ANI, which is fast and scales well to
large genome sets.

### FastANI

[FastANI](https://github.com/ParBLiSS/FastANI) computes pairwise ANI between
all passing genomes using a sketch-based approach.

### pyani

[pyani](https://github.com/widdowquinn/pyani) performs ANI analysis using the
ANIm method, producing both percentage identity and alignment coverage matrices.

For visualization, the percentage identity matrix is converted to a distance
matrix using `ani2distance-phylip.pl`, and a neighbour-joining tree is
inferred using `nj-for-dist-matrix.R` (rule `nj_tree`). The NJ tree is used
as the reference tree in the pyani ANI heatmap rather than the MLSA tree.

## Visualization and reporting

Custom scripts produce figures throughout the analysis:

- **Gene presence/absence matrix** (`table-plot.R`) — tabular PDF overview of
  locus copy numbers across all samples
- **ANI heatmap** (`tree-ANI-heatmap.R`) — ANI similarity matrix co-visualized
  with the reference phylogeny, with optional sample relabeling and automatic
  species label italicization

An interactive HTML report can be generated after a successful run:

```bash
snakemake --report report.html
```

The report embeds all outputs marked with `report(...)` in the workflow
(QUAST PDFs, gene QC matrix, ANI heatmaps) and includes a workflow graph.

## Reproducibility

The workflow is implemented in Snakemake and distributed with Conda
environments for all tools, ensuring reproducible software installation and
execution. All analyses are fully automated and reproducible from the supplied
genome assemblies, reference gene database, and configuration file.
